# Df-OS Website — SEO, GEO & AEO Rules

## 1. Purpose

This document defines SEO, GEO, and AEO execution rules for the Df-OS website.

The website must be technically crawlable, fast, structured, entity-rich, and easy for both search engines and AI answer systems to understand.

---

## 2. Core Principle

SEO, GEO, and AEO must not be treated as separate shortcuts.

The correct approach is:

> Build technically excellent SEO first, then structure content so search engines and AI systems can clearly understand the entity, context, problem, solution, audience, industry, and business outcome.

---

## 3. Df-OS Entity Definition

Use this core definition consistently:

> Df-OS is a process-first Digital Factory Operating System that helps manufacturing companies digitize people-driven shopfloor workflows, improve operational visibility, automate approvals and escalations, and connect plant-level execution with leadership decision-making.

Short definition:

> Df-OS is a modular Digital Factory Operating System for digitizing shopfloor workflows and improving manufacturing execution visibility.

Avoid describing Df-OS as only:

- MES
- ERP
- Dashboard
- Checklist app
- Generic workflow tool
- Generic no-code builder

## 5. Search Intent Rules

Every indexable page must map to one primary search intent.

| Intent | Example |
|---|---|
| Informational | What is a Digital Factory Operating System? |
| Commercial investigation | Df-OS vs MES |
| Solution-aware | shopfloor digitization software |
| Product-aware | Df-OS platform |
| Industry-specific | traceability software for FMCG |
| Problem-aware | reduce paper-based factory workflows |
| Transactional | book manufacturing software demo |

The page structure and CTA must match the intent.

## 7. URL Rules

Use clean, stable, lowercase, hyphenated URLs.

Good:

```text
/products/df-os
/products/vish-ai
/solutions/factory-control-tower
/industries/fmcg
/resources/blog/what-is-digital-factory-operating-system
```

Bad:

```text
/page1
/product-new-final
/blog/final-updated-v3
/solutions?id=123
```

If a published URL changes, create a redirect.

---

## 8. Metadata Rules

Meta titles should be specific and search-aligned.

Good examples:

```text
Digital Factory Operating System | Df-OS
Factory Control Tower Software | Df-OS
Shopfloor Digitization Platform for Manufacturers | Df-OS
Vish AI for Manufacturing Decision Intelligence | Df-OS
```

Meta descriptions should explain the page value clearly in 140–160 characters where practical.

Every indexable page must have a canonical URL.

---

## 9. Heading Rules

Each page must have one H1.

H1 should clearly define the page topic.

Good H1 examples:

```text
Digital Factory Operating System for Manufacturing Excellence
Factory Control Tower Software for Real-Time Operations Visibility
Shopfloor Digitization Platform for People-Driven Factory Workflows
```

Use H2s for major sections and H3s for subsections.

Do not use headings only for styling.

---

## 10. AEO Answer Block Rules

Important pages should include a direct answer block.

Format:

```text
Question: What is [topic]?
Answer: 50–70 word direct answer.
```

Rules:

- Answer directly.
- Avoid keyword stuffing.
- Avoid unsupported claims.
- Keep it factual.
- Make it useful for users and AI summaries.

---

## 11. SEO Summary Rules

Every major page should have a 40–70 word SEO summary.

The summary should explain:

```text
What the page is about
Who it is for
What problem it solves
What outcome it supports
```

This summary can be reused in cards, CMS previews, related content, and AI-friendly page summaries.

---

## 12. FAQ Rules

FAQ sections should be used on:

```text
Product pages
Solution pages
Industry pages
Comparison pages
Educational blogs
Glossary pages where relevant
```

FAQ schema may be added only when FAQ content is visible on the page.

Do not create fake FAQ schema.

---

## 13. Schema Rules

Use JSON-LD structured data.

Recommended schema types:

| Page Type | Schema |
|---|---|
| Homepage | Organization, WebSite, WebPage |
| Product pages | SoftwareApplication, Product, WebPage |
| Solution pages | Service, WebPage |
| Industry pages | WebPage, Service |
| Blog posts | BlogPosting, Article |
| Case studies | Article, CreativeWork |
| FAQs | FAQPage |
| Breadcrumbs | BreadcrumbList |
| Glossary pages | DefinedTerm, WebPage |

Rules:

- Schema must match visible page content.
- Do not add fake reviews, ratings, pricing, awards, or claims.
- Do not use FAQ schema without visible FAQs.
- Validate structured data before launch.

---

## 14. Internal Linking Rules

Every major page must include relevant internal links.

Required link patterns:

| Page Type | Should Link To |
|---|---|
| Product page | Solutions, industries, blogs, case studies |
| Solution page | Product, industries, related blogs, demo CTA |
| Industry page | Solutions, product, case studies, blogs |
| Blog post | Product/solution pages, related blogs, glossary |
| Case study | Related solution, industry, product, demo CTA |
| Glossary page | Related blogs, products, solutions |

Use descriptive anchor text.

Good:

```text
shopfloor digitization platform
factory control tower software
quality management workflows
```

Avoid:

```text
click here
read more
this page
```

## 20. Crawl and Indexing Rules

Index:

```text
Homepage
Product pages
Solution pages
Industry pages
Blog posts
Case studies
Glossary pages
Resource landing pages
About page
Partner page
Security/trust page
```

Usually noindex:

```text
Draft pages
Preview pages
Thank-you pages
Search result pages
Duplicate campaign pages
Internal test pages
Thin pages
```

Sitemap must include all indexable public pages and exclude drafts, archived content, noindex pages, and preview pages.

Robots.txt must reference the sitemap and must not block important public content.
